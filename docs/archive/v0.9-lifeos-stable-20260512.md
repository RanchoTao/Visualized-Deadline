## Why this version is archived

This version is an important stable checkpoint after the product shifted from a basic Visualized Deadline prototype into a broader personal operating system prototype.

It is worth preserving because it introduced a coherent product structure:

- Home as a daily control dashboard
- Dedicated Task page for full task management
- Homepage urgent deadline hotspot
- Life Tree replacing the earlier loose map concept
- Social graph plus contact list
- Review/log direction for future AI review
- Welcome-back interaction
- Local developer tools for testing reset flows

## Stable product structure

### Home

Daily control center for pressure, recommended tasks, urgent deadline area, welcome-back flow, and quick entry actions.

### Task

Core VD task system: deadline, pressure, matrix, projects, achievements, progress, and task operations.

### Life / Life Tree

Low-frequency strategic structure for life domains. This version moves away from free-form dragging and toward fixed life-tree hierarchy and focused module views.

### Social

Relationship graph plus contact-list management, with a simplified favorability-based model.

### Review

Repositioned from passive logs toward active review, leaving room for future AI-generated summaries.

### Me / Profile

Personal data, safety/export/import logic, developer testing utilities, and local reset support.

## Known issues / next directions

This version is stable enough to archive, but still has planned improvements:

- Homepage urgent area should become a cropped matrix instead of a list-like layout
- Visible LifeOS branding may later be replaced with 飞升 / 飞升科技
- Life Tree should continue evolving into a clearer hierarchical tree with second-level and third-level modules
- AI review is still a placeholder direction, not yet connected to a real model
- PWA and backend sync are future phases, not part of this stable snapshot

## Do not overwrite lightly

This branch is intended as a recovery point. Future experiments should branch from `main` or another feature branch, not directly mutate this stable archive unless intentionally creating a patched stable version.
